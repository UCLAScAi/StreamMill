IM_REL_cursor->c_del 

#include <im_db.h>
int IM_RELcursor->c_del(IM_RELC *cursorp, u_int32_t flags);
Description

The IM_RELcursor->c_del function deletes the key/data pair currently referenced by the cursor.

The flags parameter is currently unused, and must be set to 0.

To be compatible with BerkeleyDB's behavior (according to Haixun and James's code), the cursor position is automatically located at previous available data item after a delete, or set as un-initialized if the head is being deleted. This design will make IM_REL_cursor->c_get always work with DB_NEXT flag on.

Return Value

If the cursor is not yet initialized, the IM_RELcursor->c_del function will return DB_NOTFOUND. Otherwise, the IM_RELcursor->c_del function returns 0 on success.
